1

Create a GitHub Account Your free home on the internet for storing and sharing files

🐙 What is GitHub? Think of GitHub as a Google Drive for code and text files — but with superpowers. It stores your files online, keeps a history of every change you ever make, and (for our purposes) turns your files into a live public website automatically.
  1. Open your web browser (Edge, Chrome, or Firefox) and go to github.com
  2. Click the big Sign up button in the top-right corner
  3. Enter your email address, create a password, and choose a username
⚠️ Choose your username carefully! Your username becomes part of your webpage's web address. If your username is jsmith, your webpage will be at jsmith.github.io. You can change it later, but it's easier to pick a good one now — something professional, short, and memorable.
  1. GitHub will ask you to verify you're human — solve the small puzzle it shows you
  2. Check your email inbox for a verification code from GitHub and enter it on the website
  3. On the "Welcome to GitHub" screen, you can skip the optional questions by clicking Skip personalization at the bottom

Install Git The behind-the-scenes tool that sends your files to GitHub

🔄 What is Git? Git is a free program that runs invisibly on your computer. It watches the files in your project and, when you're ready, pushes (sends) them up to GitHub. Think of it like a courier that delivers your files from your desk to the internet. GitHub (the website) and Git (the program) are two different things — you need both!
  1. Go to git-scm.com/download/win — the download should start automatically
  2. Once downloaded, open the file (it will be named something like Git-2.x.x-64-bit.exe)
  3. A setup wizard will open. Click "Next" through every screen — the default settings are fine for beginners
  4. On the screen that says "Adjusting your PATH environment", make sure "Git from the command line and also from 3rd-party software" is selected (it usually is by default)
  5. Keep clicking Next and then Install
  6. When it finishes, click Finish

📸 Screenshot will appear here

Verify the installation

After installing, you can check that Git is working. Press the Windows key, type cmd, and press Enter to open a black terminal window. Then type:

git --version

You should see something like git version 2.45.0. If you do, Git is installed correctly!

✅ You're done when: git --version shows a version number in the terminal.

Install Visual Studio Code (VS Code) Your free text editor for writing and managing your webpage files

🖥️ What is VS Code? VS Code (short for "Visual Studio Code") is a free text editor made by Microsoft. It's like Notepad or Word — but designed specifically for writing code and webpage files. It has helpful features like colour-coding your text, suggesting completions as you type, and a built-in panel to send your files to GitHub.
  1. Go to code.visualstudio.com
  2. Click the big blue Download for Windows button
  3. Open the downloaded file (named something like VSCodeUserSetup-x64-1.x.x.exe)
  4. Accept the license agreement and click Next
  5. On the "Select Additional Tasks" screen, check "Add to PATH" and "Open with Code" options
  6. Click Install, then Finish — VS Code will launch automatically

Install Quarto The tool that converts your text file into a beautiful webpage

📄 What is Quarto? Quarto is a free tool that reads a simple text file you write (called a .qmd file) and converts it into a polished HTML webpage. You write in plain, readable text — Quarto handles all the technical HTML formatting for you. Think of it like Microsoft Word's "Save as PDF" but for webpages.

Download and install Quarto

  1. Go to quarto.org/docs/get-started
  2. Click the Download Quarto CLI button for Windows
  3. Open the downloaded .msi installer file
  4. Click through the installer with all default settings and click Finish

Create a Remote Repository on GitHub A "remote" repo lives on GitHub's servers — it's your webpage's permanent home online

📦 What is a Repository? A repository (or "repo" for short) is like a folder that holds all the files for one project. There are two kinds: a local repo lives on your own computer (only you can see it), and a remote repo lives on GitHub's servers (visible to anyone you choose). In this step you'll create a remote repo on GitHub — this is where your files will live on the internet and where your website gets published from.

To make GitHub publish your repo as a website, you need to give it a very specific name: yourusername.github.io. GitHub recognises this special name and automatically serves it as a live website.

  1. Sign in to github.com
  2. Click the + icon in the top-right corner and choose New repository
  3. In the "Repository name" field, type exactly: yourusername.github.io
    (replace yourusername with your actual GitHub username, all lowercase)
  4. Make sure Public is selected (so your website is visible to everyone)
  5. Check the box that says "Add a README file"
  6. Click the green Create repository button
⚠️ The name must be exact! If your username is jsmith, the repo name must be jsmith.github.io (not jsmith.GitHub.io, not jsmith.github.com). GitHub is case-sensitive for this.

Connect VS Code to Your GitHub Repository Download your repository to your computer so you can edit it

📥 What is "cloning"? Cloning a repository means making a copy of it on your computer. After cloning, you'll have a folder on your PC that is linked to GitHub — any changes you make can be sent back up to GitHub with a click. Think of it like syncing a folder with OneDrive or Dropbox.

Get your repository's address

  1. On your repository page on GitHub, click the green Code button
  2. Make sure HTTPS is selected (not SSH)
  3. Click the copy icon next to the URL — it will look like https://github.com/yourusername/yourusername.github.io.git
📸 Screenshot — Copying the repository URL from GitHub
📸 Screenshot will appear here

Clone (download) the repo in VS Code

  1. Open VS Code
  2. Press Ctrl + Shift + P to open the Command Palette (a search bar that appears at the top of VS Code)
  3. Type Git: Clone and press Enter
  4. Paste the URL you copied from GitHub and press Enter
  5. A file explorer window will appear — choose a location to save the folder (e.g., your Documents folder) and click Select Repository Location
  6. VS Code will ask "Would you like to open the cloned repository?" — click Open
  7. A pop-up may ask "Do you trust the authors of the files in this folder?" — click Yes, I trust the authors

Sign in to GitHub from VS Code

VS Code will need permission to send files to GitHub on your behalf. When prompted:

  1. A browser window will open asking you to sign in to GitHub — sign in with your username and password
  2. GitHub may show a page saying "Authorize GitHub for VS Code" — click Authorize
  3. Your browser may ask to open VS Code — click Open
✅ You're done when: VS Code shows your repository folder in the left sidebar with the README.md file listed.
7

Write Your Quarto Webpage Create a simple page using Quarto's easy text format

✍️ What is a .qmd file? A .qmd file (Quarto Markdown) is a plain text file you write with simple formatting symbols. For example, putting # before a line makes it a big heading, and surrounding text with **double asterisks** makes it bold. Quarto reads these symbols and converts them into a properly styled webpage.

Create your Quarto file

  1. In VS Code, with your repository folder open, go to File → New File
  2. Name it index.qmd and save it in your repository folder
  3. Copy and paste the following starter template into the file:
---
title: "My First Webpage"
author: "Your Name"
date: today
format:
  html:
    theme: cosmo
    toc: true
---

## Welcome!

Hello! This is my first webpage, built with [Quarto](https://quarto.org)
and hosted on GitHub Pages.

## About Me

Write a few sentences about yourself here.

## My Interests

- Interest one
- Interest two
- Interest three

## Contact

You can find me on [GitHub](https://github.com/yourusername).
📌 About the top section (the "YAML header"): The section between the --- lines at the top is called the YAML header. It gives Quarto instructions: what the title is, who wrote it, and how to format the page. The theme: cosmo line picks a visual style for your page (you can change it to flatly, journal, lux, or sketchy to try different looks). The toc: true line adds a clickable table of contents automatically.
  1. Edit the file to personalise it — change the title, add your name, write about yourself
  2. Save the file with Ctrl + S
✅ You're done when: you have an index.qmd file saved in your project folder with your personal content.
8

Render Your Page and Publish to GitHub Convert your .qmd file into HTML and send it live to the world

🚀 What is "rendering"? Rendering is the process where Quarto reads your .qmd text file and produces a polished .html webpage file. Once you have the HTML file, you'll send it to GitHub, which automatically makes it accessible on the internet.

Step 8a — Set up a Quarto project

First, tell Quarto to save the rendered HTML into a folder called docs (GitHub Pages will look there to serve your website).

  1. In VS Code, go to File → New File and name it _quarto.yml
  2. Paste the following into the file and save it:
project:
  type: website
  output-dir: docs

Step 8b — Render the page

  1. In VS Code, open the Terminal by going to Terminal → New Terminal (a black panel will appear at the bottom of VS Code)
  2. Type the following command and press Enter:
quarto render

You'll see some text scroll by. When it finishes, a new docs folder will appear in your file list containing index.html — that's your webpage!

📸 Screenshot — Running quarto render in VS Code terminal
📸 Screenshot will appear here

Step 8c — Send your files to GitHub (Git push)

Now you'll send your files from your computer up to GitHub in three steps: stage (select files), commit (save a snapshot), and push (send to GitHub).

In the VS Code terminal, type these three commands one at a time, pressing Enter after each:

git add .
git commit -m "Add my webpage"
git push

If VS Code asks you to sign in to GitHub, a browser window will open — sign in and click Authorize.

📸 Screenshot — Running git push in VS Code terminal
📸 Screenshot will appear here

Step 8d — Enable GitHub Pages

The final step: tell GitHub to publish your docs folder as a website.

  1. Go to your repository on github.com
  2. Click the Settings tab (near the top of the repository page)
  3. In the left sidebar, click Pages
  4. Under "Branch", click the dropdown that says None and select main
  5. A second dropdown will appear — select /docs
  6. Click Save

⏱️ Be patient! After clicking Save, GitHub needs a minute or two to publish your page. You'll see a blue banner that says "Your site is being built." Refresh the page after 1–2 minutes and the banner will change to green with your live URL.
✅ You're done when: the GitHub Pages section shows a green banner with the text "Your site is live at https://yourusername.github.io".

Your webpage is now publicly accessible at:

https://yourusername.github.io

Replace yourusername with your actual GitHub username. You can open this URL in any browser — on your phone, your computer, or share it with anyone in the world. No login required to view it.

✅ How to find your URL from GitHub: Go to your repository on github.com → click the Settings tab → click Pages in the left sidebar. GitHub will show your live URL in a green box. You can click it directly from there.

What to do next

Update your page — Edit your index.qmd file in VS Code, then run quarto render and the three git commands again. Your live website will update within a few minutes.

Try different themes — In your index.qmd, change theme: cosmo to one of these: flatly, lux, journal, minty, sandstone, or sketchy. Render again to see the difference.

Add images — Put an image file (e.g., photo.jpg) in your project folder and add this to your .qmd file:

![A caption for your image](photo.jpg)

Learn more Quarto features — The Quarto documentation at quarto.org/docs/websites has guides on adding navigation menus, multiple pages, and more.
